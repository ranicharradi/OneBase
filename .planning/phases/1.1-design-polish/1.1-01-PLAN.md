<task type="auto">
  <name>Task 1: Design system overhaul — fonts, colors, animations, utilities</name>
  <files>frontend/index.html, frontend/src/app.css</files>
  <action>
BEFORE CODING: Load the frontend-design skill at `.agents/skills/frontend-design/SKILL.md`. Follow its Design Thinking process — understand context, choose aesthetic direction, identify what makes this UNFORGETTABLE.

**Aesthetic direction guidance:** This is a serious data management platform for enterprise staff. Appropriate directions: refined dark industrial, dark editorial, luxury dark, or brutalist-precision. NOT playful, NOT pastel, NOT maximalist chaos. The design should feel like a premium tool — controlled, precise, atmospheric.

**index.html changes:**
1. Add Google Fonts (or Bunny Fonts) `<link>` tags for TWO fonts: one distinctive display/heading font and one refined body font
2. Choose fonts that are NOT: Inter, Roboto, DM Sans, Space Grotesk, Open Sans, Nunito, Poppins, or any other overused AI-default font
3. Good candidates for this context: Instrument Sans, Plus Jakarta Sans, Satoshi, General Sans, Switzer, Cabinet Grotesk, Outfit, Syne, Manrope — but pick what serves the aesthetic direction. Pair differently (e.g., geometric display + humanist body, or vice versa)
4. Add a `<link rel="preconnect">` for the font CDN for performance

**app.css changes:**
1. Update `@theme` with new font families matching the imported fonts
2. Refine the color palette — keep the dark surface scale but give the accent color personality. Options: shift from generic blue-500 to a distinctive tone (teal/cyan, amber/gold, emerald, rose, or a custom hue). Add a secondary accent if it serves the design.
3. Add at minimum these animation keyframes (6+ total):
   - `fadeIn` (refine existing)
   - `slideUp` (refine existing)
   - `slideInLeft` (for sidebar nav items)
   - `pulse-glow` (for active/processing states)
   - `shimmer` (for loading skeletons — better than plain pulse)
   - `float` (subtle hover lift effect)
   - Any others the aesthetic direction demands
4. Add utility CSS classes for atmospheric effects:
   - A subtle noise/grain texture overlay (using CSS `background-image` with inline SVG or gradient noise)
   - A glass/frosted effect utility class
   - Glow/halo effects for accent elements
5. Refine global styles: selection color, scrollbar styling, focus ring styles
6. Add `@layer components` with reusable card, button, and input base styles if they reduce repetition

**CRITICAL**: Every design token change here cascades to ALL other frontend files. Choose wisely — Plans 02 and 03 will apply this system.

<task type="auto">
  <name>Task 2: Redesign app shell and login page</name>
  <files>frontend/src/components/Layout.tsx, frontend/src/pages/Login.tsx</files>
  <action>
Apply the design system from Task 1 to the two most visible UI surfaces.

**Layout.tsx redesign:**
1. Sidebar: Add visual depth — subtle gradient background, noise texture overlay, or atmospheric blur. NOT flat surface-900.
2. Brand area: Make the OneBase logo/wordmark distinctive. Consider: larger typography, accent glow, custom SVG mark, or typographic treatment.
3. Nav items: Add micro-interactions on hover (translate, scale, glow, border animation). Active state should feel emphatic, not just "slightly blue". Consider using `slideInLeft` animation on mount with staggered delays.
4. User footer: Make the avatar more interesting (gradient border, glow ring, or distinctive shape). Add a subtle transition on the logout button.
5. Mobile header: Ensure the hamburger menu trigger has personality matching the sidebar.
6. Content area: Consider a subtle texture or gradient on the main content background to create depth between sidebar and content.
7. Add transition animations when the sidebar opens/closes on mobile.

**Login.tsx redesign:**
1. This is the FIRST thing users see — it must be memorable. Not just "dark background with centered card."
2. Background: Create atmosphere — layered gradients, animated grain, geometric patterns, mesh gradients, or subtle animated shapes. The background should have DEPTH.
3. Logo/brand: Make it hero-level. Large, bold, distinctive. The OneBase name should feel like it belongs to THIS application, not a generic SaaS template.
4. Form card: Consider alternatives to the standard rounded-2xl card — maybe no visible card (inputs floating on atmosphere), or a card with distinctive border treatment (gradient borders, glow edges), or asymmetric positioning.
5. Input fields: Style distinctively — custom focus states with animation, not just ring color change. Consider underline-style inputs, or inputs with gradient borders on focus.
6. Submit button: Needs presence — hover animation, gradient fill, glow effect, or scale transform. Not just "bg-accent hover:bg-accent-darker".
7. Add stagger animation to elements appearing on page load (logo → tagline → form → inputs → button, each with animation-delay).
8. Footer text: Use the display font with distinctive styling, not plain text-xs.

**Preserve all functionality.** Do NOT change any state management, API calls, form handlers, routing, or event handlers. Only change JSX structure, Tailwind classes, and inline styles.
