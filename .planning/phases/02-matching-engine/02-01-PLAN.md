1. **Create `backend/app/services/blocking.py`:**
   - `text_block(db: Session, source_ids: list[int]) -> set[tuple[int, int]]`:
     - Query all active StagedSuppliers for given source_ids
     - Build prefix dict: key = normalized_name[:3], value = list of supplier IDs (only if normalized_name length >= 3)
     - Build first-token dict: key = normalized_name.split()[0], value = list of supplier IDs
     - For each bucket with suppliers from different sources, generate cross-entity pairs
     - Normalize pairs: always (min_id, max_id) to avoid duplicates
     - Return deduplicated set of pairs
   - `embedding_block(db: Session, source_ids: list[int], k: int = None) -> set[tuple[int, int]]`:
     - k defaults to `settings.matching_blocking_k`
     - For each active supplier with an embedding, query pgvector for K nearest neighbors: `select(StagedSupplier).filter(StagedSupplier.data_source_id != supplier.data_source_id, StagedSupplier.status == "active").order_by(StagedSupplier.name_embedding.cosine_distance(supplier.name_embedding)).limit(k)`
     - Collect cross-entity pairs (min_id, max_id)
     - Return deduplicated set
   - `combine_blocks(*block_results: set[tuple[int, int]]) -> set[tuple[int, int]]`: Union of all blocking results

2. **Create `backend/app/services/scoring.py`:**
   - `score_pair(supplier_a: StagedSupplier, supplier_b: StagedSupplier) -> dict`:
     - Import: `from rapidfuzz.distance import JaroWinkler` and `from rapidfuzz import fuzz`
     - `jaro_winkler`: `JaroWinkler.similarity(a.normalized_name or "", b.normalized_name or "")` — returns 0-1
     - `token_jaccard`: `fuzz.token_set_ratio(a.normalized_name or "", b.normalized_name or "") / 100.0` — token_set_ratio returns 0-100
     - `embedding_cosine`: If both have embeddings, compute `1.0 - cosine_distance(a.name_embedding, b.name_embedding)` using numpy dot product (embeddings are L2-normalized so dot = cosine sim). If either is None, return 0.5 (neutral).
     - `short_name_match`: Both non-None and equal → 1.0; both non-None and different → 0.0; either None → 0.5
     - `currency_match`: Both non-None and equal (case-insensitive) → 1.0; both non-None and different → 0.0; either None → 0.5
     - `contact_match`: Both non-None → `JaroWinkler.similarity(a.contact_name, b.contact_name)`; either None → 0.5
     - Compute weighted confidence: `sum(signal_value * weight for signal, weight)` using settings weights
     - Return `{"confidence": confidence, "signals": {"jaro_winkler": ..., "token_jaccard": ..., "embedding_cosine": ..., "short_name_match": ..., "currency_match": ..., "contact_match": ...}}`

3. **Create `backend/app/services/clustering.py`:**
   - `find_groups(pairs: list[tuple[int, int]], max_cluster_size: int = None) -> list[set[int]]`:
     - max_cluster_size defaults to `settings.matching_max_cluster_size`
     - Implement Union-Find (disjoint set) with path compression and union by rank
     - For each pair (a, b), union(a, b)
     - Collect connected components by grouping by find(x) for all nodes
     - For groups exceeding max_cluster_size: log warning with group size, but keep the group intact (flag it — return it with a note). The group is still valid, just large.
     - Return list of sets, each set containing supplier IDs in the group

   For SQLite tests, create StagedSupplier fixtures with data_source_id and normalized_name set. The name_embedding column uses LargeBinary fallback in SQLite, so embedding_block tests should mock the pgvector query path and test the pair-assembly logic separately.
