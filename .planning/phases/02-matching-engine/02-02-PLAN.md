1. **Create `backend/app/services/matching.py`:**
   ```python
   def run_matching_pipeline(
       db: Session,
       batch_id: int,
       progress_callback: Callable[[str, int], None] | None = None,
       invalidate_source_id: int | None = None,
   ) -> dict:
   ```
   Implementation steps:
   a. If `invalidate_source_id` is set: update all MatchCandidates where supplier_a or supplier_b belongs to that source to status="invalidated". Also delete their MatchGroup assignments.
   b. Get all active source_ids from the import batch's data source and all other sources that have active suppliers.
   c. If fewer than 2 source_ids, return {"candidate_count": 0, "group_count": 0} (need cross-entity).
   d. **BLOCKING** stage: Call `text_block(db, source_ids)` and `embedding_block(db, source_ids)`, then `combine_blocks()`. Report progress via callback.
   e. **SCORING** stage: For each pair, load supplier objects, call `score_pair()`. Filter by `settings.matching_confidence_threshold`. Report progress.
   f. **CLUSTERING** stage: Call `find_groups()` on above-threshold pairs. Report progress.
   g. **INSERTING** stage: Create MatchGroup records for each group. Create MatchCandidate records with confidence, match_signals (the signals dict), group_id, status="pending". Use bulk operations where possible. Normalize pair ordering (min_id, max_id). Handle unique constraint — skip duplicates with non-invalidated status. Report progress.
   h. db.flush() (caller manages commit)
   i. Return {"candidate_count": N, "group_count": M}

2. **Replace stub in `backend/app/tasks/matching.py`:**
   - Make task `bind=True` for progress reporting: `@celery_app.task(bind=True, name="run_matching")`
   - Accept `batch_id: int` and optional `invalidate_source_id: int = None`
   - Create SessionLocal, wrap in try/finally
   - Store matching_task_id on ImportBatch before running
   - Define progress_callback that calls `self.update_state(state=stage.upper(), meta={"stage": stage, "progress": pct})`

1. **Create `backend/app/services/retraining.py`:**
   - `retrain_weights(db: Session) -> dict | None`:
     - Query MatchCandidates with status in ("confirmed", "rejected") and non-null match_signals
     - If count < 20, return None
     - Build feature matrix X (6 signal values per candidate) and labels y (1 for confirmed, 0 for rejected)
     - Fit `sklearn.linear_model.LogisticRegression` (import at function level to keep it optional — if sklearn not available, fall back to manual normalized absolute coefficients)
     - Actually, to avoid adding sklearn dependency: implement simple logistic regression using numpy. Or even simpler: compute average signal values for confirmed vs rejected, derive weights from the signal discriminative power (ratio of mean_confirmed / mean_all for each signal, normalized to sum=1).
     - **Simpler approach (no sklearn):** For each signal, compute the mean value in confirmed candidates minus mean in rejected. Signals with larger positive difference get higher weight. Normalize to sum=1.0 and clamp to [0.01, 0.5] range.
     - Return dict: `{"jaro_winkler": w1, "token_jaccard": w2, ...}`
