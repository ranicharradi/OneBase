1. **Create `backend/app/services/notifications.py`:**
   - `publish_notification(notification_type: str, data: dict) -> None`:
     - Import redis (already in requirements.txt): `import redis`
     - Create Redis client from `settings.redis_url`
     - Publish JSON message to channel `"onebase:notifications"`:
       ```python
       message = json.dumps({"type": notification_type, "data": data, "timestamp": datetime.utcnow().isoformat()})
       r.publish("onebase:notifications", message)
       ```
   - Use a module-level Redis connection (lazy singleton) to avoid creating new connections per call
   - Handle Redis connection errors gracefully (log warning, don't crash the task)

2. **Create `backend/app/routers/ws.py`:**
   - `router = APIRouter()` (no prefix — WebSocket paths are different)
   - WebSocket endpoint at `/ws/notifications`:
     ```python
     @router.websocket("/ws/notifications")
     async def ws_notifications(websocket: WebSocket):
     ```
   - Accept connection, then subscribe to Redis pub/sub channel `"onebase:notifications"`:
     - Use `redis.asyncio` (or run sync Redis subscription in a thread via `asyncio.to_thread`)
     - Actually, use `redis` sync client in a background thread since the rest of the backend is sync SQLAlchemy
     - Simpler approach: Use `aioredis`/`redis.asyncio` just for the WebSocket endpoint (FastAPI WebSocket handlers are async naturally)
     - **Recommended approach:** Use `redis.asyncio.from_url(settings.redis_url)` for the async pub/sub listener in the WS handler:
       ```python
       import redis.asyncio as aioredis
       async_redis = aioredis.from_url(settings.redis_url)
       pubsub = async_redis.pubsub()
       await pubsub.subscribe("onebase:notifications")
       try:
           async for message in pubsub.listen():
               if message["type"] == "message":
                   await websocket.send_text(message["data"].decode())
       except WebSocketDisconnect:
           pass
       finally:
           await pubsub.unsubscribe("onebase:notifications")
           await async_redis.close()
       ```
   - Handle WebSocketDisconnect gracefully
   - No auth required for WebSocket (JWT validation on WS is complex — acceptable for v1, notifications are non-sensitive)

   - On failure: call `publish_notification("matching_failed", {"batch_id": batch_id, "error": str(e)})`
   - Import: `from app.services.notifications import publish_notification`
