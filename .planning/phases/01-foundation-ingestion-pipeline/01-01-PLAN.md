    **backend/app/schemas/auth.py** — Pydantic v2 models:
    - `LoginRequest`: username (str), password (str)
    - `TokenResponse`: access_token (str), token_type (str, default "bearer")
    - `UserCreate`: username (str, min_length=3, max_length=100), password (str, min_length=6)
    - `UserResponse`: id (int), username (str), is_active (bool), created_at (datetime). Use `model_config = ConfigDict(from_attributes=True)`.

    **backend/app/services/auth.py** — Stateless auth functions:
    - `hash_password(password: str) -> str`: Use passlib CryptContext with bcrypt
    - `verify_password(plain: str, hashed: str) -> bool`: Verify with CryptContext
    - `create_token(username: str) -> str`: PyJWT encode with sub=username, exp=now+jwt_expire_minutes, iat=now
    - `authenticate_user(db: Session, username: str, password: str) -> User | None`: Query user, verify password
    - `create_initial_user(db: Session)`: If admin_username and admin_password are set in config AND no users exist in DB, create the admin user. Called on app startup.

    **backend/app/services/audit.py** — Simple audit helper:
    - `log_action(db: Session, user_id: int, action: str, entity_type: str = None, entity_id: int = None, details: dict = None)`: Create AuditLog record. Flush (not commit — caller controls transaction).

    **backend/app/dependencies.py** — Add `get_current_user` dependency:
    - Use `OAuth2PasswordBearer(tokenUrl="/api/auth/login")`
    - Decode JWT token, extract username from "sub" claim
    - Query user from DB
    - Raise HTTPException(401) if token invalid or user not found
    - Return User object

    **backend/app/routers/auth.py** — Endpoints:
    - `POST /api/auth/login`: Accept `OAuth2PasswordRequestForm`, authenticate, create token, log "login" action, return TokenResponse
    - `POST /api/auth/users`: Accept UserCreate (requires get_current_user), hash password, create user, log "create_user" action, return UserResponse
    - `GET /api/auth/me`: Return current user (requires get_current_user)

    **backend/app/routers/users.py** — Endpoints:
    - `GET /api/users`: List all users (requires get_current_user), return list of UserResponse

    **backend/app/main.py** — Update to:
    - Include auth and users routers with prefix="/api"
    - Add lifespan context manager that calls `create_initial_user(db)` on startup
    - Log startup message

    **Tests** — Write tests FIRST (TDD red phase), then implement:

    - Use conftest.py fixtures for test client and test DB
