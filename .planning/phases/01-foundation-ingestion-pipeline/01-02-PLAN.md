<objective>
Build the complete ingestion pipeline backend: CSV parsing, data source CRUD, file upload, name normalization, embedding computation, re-upload supersession, and Celery task orchestration with progress tracking.

Purpose: This is the core data pipeline that processes supplier CSV exports from raw files into normalized, embedded staging records. Without this, there's no data to match or review.
Output: Working API endpoints for data source management and file upload, Celery worker that processes uploads through the full pipeline (parse → map → store → normalize → embed), re-upload lifecycle with supersession, and a matching stub task.
</objective>

    **backend/app/schemas/upload.py** — Pydantic v2 schemas:
    - `UploadResponse`: batch_id (int), task_id (str), filename (str), message (str)
    - `BatchResponse`: id (int), data_source_id (int), filename (str), uploaded_by (str), row_count (int | None), status (str), error_message (str | None), created_at (datetime), task_id (str | None). Use ConfigDict(from_attributes=True).
    - `TaskStatusResponse`: task_id (str), state (str), stage (str | None), progress (int | None), detail (str | None), row_count (int | None)

    **backend/app/services/ingestion.py** — Ingestion orchestration service:
    - `run_ingestion(db: Session, batch_id: int, file_content: bytes, progress_callback: Callable | None = None) -> int`:
      1. Load ImportBatch and its DataSource from DB
      2. **PARSE**: Call parse_csv(file_content, delimiter=source.delimiter). Call progress_callback('parsing', 0) if provided.
      3. **MAP**: For each row, apply column_mapping to extract key fields (supplier_name → name, supplier_code → source_code, etc.). Store entire row as raw_data JSONB.
      4. **SUPERSEDE** (if re-upload): Query existing active StagedSuppliers for this data_source_id. Mark all as status='superseded'. Query MatchCandidates where supplier_a_id or supplier_b_id references any superseded record AND status='pending' → update to status='invalidated'. Count invalidated for reporting.
      5. **STORE**: Create StagedSupplier records with extracted key fields + raw_data + import_batch_id + data_source_id. Batch insert (add_all). Call progress_callback('normalizing', 33).
      6. **NORMALIZE**: For each staged supplier, compute normalized_name = normalize_name(supplier.name). Update records. Call progress_callback('normalizing', 50).
      7. **EMBED**: Collect all normalized_names, call compute_embeddings(names). Update each record's name_embedding. Call progress_callback('embedding', 66).
      8. **FINALIZE**: Update batch.row_count and batch.status='completed'. Call progress_callback('complete', 100).
      9. Return row_count.
    - On error: Set batch.status='failed', batch.error_message=str(error), re-raise.

    **backend/app/tasks/ingestion.py** — Celery task:
    - `@celery_app.task(bind=True, name='process_upload')` 
    - `process_upload(self, batch_id: int)`:
      1. Create a SessionLocal() session (Celery tasks manage their own sessions).
      2. Load batch, read file from data/uploads/{batch.filename}.
      3. Define progress_callback that calls self.update_state(state=stage.upper(), meta={'stage': stage, 'progress': pct}).
      4. Call run_ingestion(db, batch_id, file_content, progress_callback).
      5. On success: commit, then enqueue matching stub: `run_matching.delay(batch_id)`.
      6. On error: rollback, set batch status to failed, commit, re-raise.
      7. Always: close session.

    **backend/app/tasks/matching.py** — Matching stub:
    - `@celery_app.task(name='run_matching')`
    - `run_matching(batch_id: int)`: Log "Matching stub for batch {batch_id}". Return {'status': 'stub', 'batch_id': batch_id}. (Real matching built in Phase 2.)
