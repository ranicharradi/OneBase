Auth endpoints (Plan 01):
- POST /api/auth/login — body: {username, password} via OAuth2PasswordRequestForm → {access_token, token_type}
- GET /api/auth/me — requires Bearer token → {id, username, is_active, created_at}
- POST /api/auth/users — requires Bearer token, body: {username, password} → {id, username, is_active, created_at}
- GET /api/users — requires Bearer token → [{id, username, is_active, created_at}]

Source endpoints (Plan 02):
- POST /api/sources — requires Bearer token, body: {name, description?, file_format?, delimiter?, column_mapping: {supplier_name, supplier_code, short_name?, currency?, payment_terms?, contact_name?, supplier_type?}} → DataSourceResponse
- GET /api/sources — requires Bearer token → [DataSourceResponse]
- GET /api/sources/{id} — requires Bearer token → DataSourceResponse
- PUT /api/sources/{id} — requires Bearer token → DataSourceResponse
- DELETE /api/sources/{id} — requires Bearer token → 204

DataSourceResponse: {id, name, description, file_format, delimiter, column_mapping, created_at, updated_at}

    **IMPORTANT:** Use the `frontend-design` skill for ALL React component creation. Every page must be production-grade with dark theme.

    **Project setup** — Initialize Vite React-TS project:
    Run `npm create vite@latest frontend -- --template react-ts` if not already scaffolded, or manually create files. Install dependencies:
    ```
    npm install react-router @tanstack/react-query @tanstack/react-table
    npm install -D tailwindcss @tailwindcss/vite
    ```

    **frontend/vite.config.ts** — Vite config with Tailwind CSS 4 plugin and API proxy:
    ```typescript
    import { defineConfig } from 'vite'
    import react from '@vitejs/plugin-react'
    import tailwindcss from '@tailwindcss/vite'

    export default defineConfig({
      plugins: [react(), tailwindcss()],
      server: {
        proxy: {
          '/api': {
            target: 'http://localhost:8000',
            changeOrigin: true,
          },
        },
      },
    })
    ```

    **frontend/src/app.css** — Tailwind CSS 4 entry (CSS-first config, NO tailwind.config.js):
    ```css
    @import "tailwindcss";

    @theme {
      --color-surface-900: #0f172a;
      --color-surface-800: #1e293b;
      --color-surface-700: #334155;
      --color-surface-600: #475569;
      --color-primary-500: #3b82f6;
      --color-primary-600: #2563eb;
      --color-primary-400: #60a5fa;
    }
    ```
    Note: Tailwind CSS 4 uses `@import "tailwindcss"` (not `@tailwind base/components/utilities`). Theme customization via `@theme` directive in CSS (not tailwind.config.js). The frontend-design skill will guide the visual design.

    **frontend/src/api/types.ts** — TypeScript types matching backend schemas:
    - `User`: { id: number, username: string, is_active: boolean, created_at: string }
    - `TokenResponse`: { access_token: string, token_type: string }
    - `DataSource`: { id: number, name: string, description: string | null, file_format: string, delimiter: string, column_mapping: ColumnMapping, created_at: string, updated_at: string }
    - `ColumnMapping`: { supplier_name: string, supplier_code: string, short_name?: string, currency?: string, payment_terms?: string, contact_name?: string, supplier_type?: string }
    - `DataSourceCreate`: { name: string, description?: string, file_format?: string, delimiter?: string, column_mapping: ColumnMapping }
    - `UserCreate`: { username: string, password: string }

    **frontend/src/api/client.ts** — Typed API client:
    - `async function request<T>(path: string, options?: RequestInit): Promise<T>`: Attaches JWT from localStorage as Bearer token. Handles 401 by clearing token and redirecting to /login. Throws on non-OK responses.
    - Export convenience methods: `api.get<T>(path)`, `api.post<T>(path, body)`, `api.put<T>(path, body)`, `api.delete(path)`, `api.upload<T>(path, formData)` (for file uploads — no Content-Type header, let browser set multipart boundary).

    **frontend/src/hooks/useAuth.ts** — Auth context + hook:
    - `AuthProvider`: React context provider. Stores token in localStorage. On mount, check if token exists and validate via GET /api/auth/me. Provides: user (User | null), login(username, password), logout(), isLoading, isAuthenticated.
    - `useAuth()`: Hook to access auth context.
    - Login function: POST /api/auth/login with OAuth2-compatible form body (application/x-www-form-urlencoded with `username` and `password` fields — FastAPI OAuth2PasswordRequestForm expects form data, not JSON).

    **frontend/src/components/ProtectedRoute.tsx** — Route guard:
    - If auth loading: show skeleton/spinner
    - If not authenticated: redirect to /login
    - If authenticated: render children

    **frontend/src/components/Layout.tsx** — App shell (USE frontend-design SKILL):
    - Dark theme background (slate-900/950)
    - Left sidebar with navigation links: Upload (link to /upload), Sources (/sources), Users (/users)
    - App title "OneBase" at top of sidebar
    - Main content area with `<Outlet />` from React Router
    - Current user display + logout button in sidebar footer
    - Responsive: sidebar collapses on mobile

    **frontend/src/pages/Login.tsx** — Login page (USE frontend-design SKILL):
    - Dark themed, centered card with OneBase branding
    - Username + password fields
    - Submit button with loading state
    - Error message display for invalid credentials
    - Redirects to / on successful login

    **frontend/src/App.tsx** — Router setup:
    - React Router with routes: /login (public), / → Layout with nested routes: /sources, /users, /upload (placeholder)
    - Default redirect from / to /sources
    - Wrap in AuthProvider and QueryClientProvider (TanStack Query)

    **frontend/src/main.tsx** — Entry point: render App into #root

    **frontend/index.html** — Standard Vite HTML with dark background (`<body class="bg-slate-950 text-gray-100">`)

    **frontend/Dockerfile** — Multi-stage build for production:
    ```dockerfile
    FROM node:22-alpine AS build
    WORKDIR /app
    COPY package*.json ./
    RUN npm ci
    COPY . .
    RUN npm run build

    FROM nginx:alpine
    COPY --from=build /app/dist /usr/share/nginx/html
    COPY nginx.conf /etc/nginx/conf.d/default.conf
    ```
    Note: nginx.conf for production will be created when needed. Dev uses Vite proxy.
